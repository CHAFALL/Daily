Listen Server Vs Dedicated Server

1. Listen Server
Listen Server는 게임을 플레이하는 클라이언트 중 하나가 동시에 서버 역할을 하는 형태입니다. 이 경우 서버는 게임을 실행하는 플레이어의 컴퓨터에서 직접 호스팅됩니다.

호스팅 방법: 플레이어 중 한 명이 자신의 컴퓨터에서 서버를 실행하고 다른 플레이어들이 해당 서버에 접속합니다.
장점:
설정이 간단하고 추가 서버 비용이 들지 않습니다.
개인적인 소규모 게임이나 친구들끼리 하는 게임에 적합합니다.
플레이어가 게임을 만들면 바로 그 게임의 서버가 되기 때문에 빠르게 게임을 시작할 수 있습니다.
단점:
서버를 호스팅하는 플레이어의 컴퓨터 성능에 의존하기 때문에 성능 문제나 네트워크 연결 문제가 발생할 수 있습니다.
서버 호스트가 게임을 종료하거나 문제가 발생하면 모든 연결이 끊기고 게임이 종료됩니다.
보안성이 낮아, 악의적인 클라이언트에 의해 게임이 조작될 가능성이 있습니다.
일반적으로 많은 수의 플레이어를 지원하지 못합니다.

2. Dedicated Server
Dedicated Server는 클라이언트로 사용되지 않는 전용 서버입니다. 이 서버는 게임을 호스팅하는 데만 사용되고, 보통 고성능 컴퓨터 또는 클라우드 서버에서 운영됩니다.

호스팅 방법: 서버 전용 컴퓨터에서 게임 서버 소프트웨어를 실행하여, 플레이어들이 이 서버에 접속합니다.
장점:
서버는 고성능 하드웨어와 안정적인 네트워크 연결을 사용할 수 있으므로 더 많은 플레이어를 지원하고 더 안정적인 게임 경험을 제공합니다.
서버가 클라이언트와 독립적이기 때문에, 호스트가 게임을 종료하더라도 서버는 계속 운영됩니다.
게임의 공정성을 유지하고 보안성을 높이기 위해 서버에서 모든 게임 로직을 처리할 수 있습니다.
지속적으로 운영되는 서버로, 랭킹 시스템, 매치메이킹, 전적 저장 등 다양한 기능을 안정적으로 제공합니다.
단점:
서버를 운영하기 위한 추가적인 비용이 발생합니다.
서버 설정 및 유지 관리가 필요하므로 기술적인 지식이 요구될 수 있습니다.

요약
Listen Server는 플레이어 중 한 명이 게임과 서버 역할을 동시에 하는 방식으로, 설정이 간편하고 비용이 들지 않지만, 안정성과 성능이 떨어질 수 있습니다.
Dedicated Server는 서버 전용 컴퓨터에서 게임을 호스팅하는 방식으로, 안정적이고 많은 플레이어를 지원할 수 있으며 보안성이 높지만, 추가 비용이 발생하고 설정이 더 복잡할 수 있습니다.
이 두 서버 유형은 각각의 장단점이 있으므로, 게임의 규모, 예산, 플레이어 수, 필요한 성능 및 안정성 등을 고려하여 적절한 서버 유형을 선택하는 것이 중요합니다.

-------

Listen Server가 렌더링 그래픽스를 수행해야 하는 이유는 이 서버가 서버 역할을 하면서 동시에 게임을 플레이하는 클라이언트 역할도 하기 때문입니다. 좀 더 구체적으로 설명하자면:

1. 플레이어와 서버의 동시 역할
Listen Server는 게임을 호스팅하는 플레이어의 컴퓨터에서 서버와 클라이언트가 동시에 실행됩니다. 이 컴퓨터는 서버로서 게임의 논리적 처리와 네트워크 관리 작업을 수행하는 동시에, 클라이언트로서 게임 화면을 플레이어에게 보여주기 위해 그래픽을 렌더링합니다.

클라이언트 기능: 플레이어는 게임을 플레이하고 화면에 그래픽을 렌더링하여 게임 세계를 볼 수 있어야 합니다. 이 과정에서 플레이어의 컴퓨터는 일반 클라이언트처럼 그래픽을 렌더링해야 합니다.

서버 기능: 동시에 이 컴퓨터는 다른 플레이어들에게 게임 데이터를 전달하고, 다른 클라이언트에서 보내는 데이터를 받아들이는 서버 역할도 수행합니다.

2. 그래픽 렌더링의 필요성
Listen Server가 그래픽을 렌더링하는 이유는 다음과 같습니다:

플레이어의 게임 경험: Listen Server의 호스트가 되는 플레이어는 게임을 플레이하고 있으므로, 이 플레이어가 게임 화면을 보고 조작할 수 있도록 그래픽이 렌더링되어야 합니다.

하드웨어 자원 활용: Listen Server는 플레이어의 개인 컴퓨터에서 실행되기 때문에, 이 컴퓨터는 일반적으로 GPU(그래픽 처리 장치)를 포함하고 있습니다. 따라서 클라이언트 역할을 수행하면서 GPU를 활용하여 게임 화면을 렌더링하게 됩니다.

3. Dedicated Server와의 차이점
이와는 달리, Dedicated Server는 오로지 서버 역할만을 수행합니다. 게임 플레이어로서의 기능은 없기 때문에 그래픽 렌더링이 필요하지 않습니다. 대신, 이 서버는 게임의 논리 처리, 물리 엔진, 네트워크 관리 등 게임의 서버 관련 작업에 집중하게 됩니다. Dedicated Server는 보통 모니터나 GPU가 필요 없으며, 콘솔 환경에서 실행되거나 명령줄에서만 작동할 수 있는 경우가 많습니다.

결론
Listen Server는 플레이어가 게임을 플레이하면서 동시에 서버 역할을 수행하기 때문에, 그래픽 렌더링이 필요합니다. 반면, Dedicated Server는 순수하게 서버 기능만 수행하므로 그래픽 렌더링이 필요 없습니다. 이러한 차이는 두 서버의 기능과 용도에 기인합니다.


-----
Listen Server -> 호스트는 자기가 서버이기 때문에 데이터 전송하는 데 걸리는 연관된 지연 시간이 없음, 반면에 다른 클라이언트들은 있음 (공평 x) - 그래서 소규모 게임에서만 쓰는 듯.
